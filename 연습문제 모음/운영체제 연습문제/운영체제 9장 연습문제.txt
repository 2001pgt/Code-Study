운영체제 9장 연습문제

1.페이징의 기본 개념
페이징에서 “페이지(page)”와 “프레임(frame)”의 관계로 옳은 것은?
a) 페이지는 물리 메모리, 프레임은 논리 주소 공간을 나눈 단위이다.
b) 페이지와 프레임은 크기가 다르게 설정된다.
c) 프로세스의 논리 주소 공간은 페이지 단위로, 물리 메모리는 동일 크기의 프레임 단위로 나눈다.
d) 페이지는 가변 크기, 프레임은 고정 크기이다.

(참조: 9장 페이징 메모리 관리.pdf, pp.4–5)

2.페이징의 장점
페이징 기법의 장점으로 옳지 않은 것은?
a) 외부 단편화가 완전히 제거된다.
b) 페이지 크기를 시스템마다 다르게 설정할 수 있어 이식성이 높다.
c) 페이지 테이블을 항상 메모리 상에서만 관리하므로 디스크 접근이 불필요하다.
d) 고정 크기 단위 분할로 구현이 비교적 간단하다.

(참조: 9장 페이징 메모리 관리.pdf, p.6)

3.논리 주소 구조
32비트 논리 주소에서 페이지 크기가 4KB(2¹²바이트)일 때, 페이지 번호와 옵셋(offset)의 비트 분할로 옳은 것은?
a) 상위 12비트: 페이지 번호, 하위 20비트: 옵셋
b) 상위 20비트: 페이지 번호, 하위 12비트: 옵셋
c) 상위 16비트: 페이지 번호, 하위 16비트: 옵셋
d) 상위 24비트: 페이지 번호, 하위 8비트: 옵셋

(참조: 9장 페이징 메모리 관리.pdf, p.17)

4.페이지 테이블 크기 계산
32비트 시스템, 페이지 크기 4KB, 페이지 테이블 엔트리 크기 4바이트일 때, 한 프로세스의 전체 페이지 테이블 크기는 얼마인가?
a) 1 MB
b) 2 MB
c) 4 MB
d) 8 MB

(참조: 9장 페이징 메모리 관리.pdf, p.13)

5.내부 단편화
페이징에서 내부 단편화가 발생하는 이유로 가장 적절한 것은?
a) 프로세스들 간 프레임 크기가 달라서
b) 마지막 페이지에 요청 크기보다 작은 남은 공간이 생기기 때문에
c) 페이지 테이블이 메모리를 많이 차지하기 때문에
d) MMU가 논리 주소를 물리 주소로 변환할 때 추가 공간이 필요하기 때문에

(참조: 9장 페이징 메모리 관리.pdf, p.14)

6.주소 변환 과정
페이징에서 CPU가 논리 주소를 물리 주소로 변환할 때, MMU가 참조하는 구조가 아닌 것은?
a) Page Table Base Register (PTBR)
b) 페이지 테이블 엔트리
c) TLB (Translation Lookaside Buffer)
d) 세그먼트 레지스터

(참조: 9장 페이징 메모리 관리.pdf, pp.19–20)

7.TLB 히트와 미스
TLB(Translation Lookaside Buffer)의 동작으로 옳은 설명은?
a) TLB 히트 시에도 페이지 테이블을 항상 한 번은 참조한다.
b) TLB 미스 시 MMU가 페이지 테이블에서 엔트리를 읽어와 TLB에 삽입한다.
c) TLB는 모든 페이지 번호를 저장할 수 있도록 매우 크게 설계된다.
d) TLB는 페이지 번호가 아닌 물리 프레임 번호만 캐싱한다.

(참조: 9장 페이징 메모리 관리.pdf, pp.27–28)

8.TLB 성능
TLB reach(도달 범위)에 해당하는 것은?
a) 페이지 크기 ÷ TLB 엔트리 수
b) 페이지 크기 × TLB 엔트리 수
c) TLB 엔트리 수 ÷ 페이지 크기
d) (페이지 크기 + TLB 엔트리 수) ÷ 2

(참조: 9장 페이징 메모리 관리.pdf, p.33)

9.컨텍스트 스위칭 시 TLB 처리
서로 다른 프로세스로 컨텍스트 스위칭할 때 OS가 TLB에 대해 수행해야 할 작업은?
a) 아무 작업도 하지 않는다.
b) TLB의 모든 엔트리를 invalidate(지우기) 한다.
c) 남은 엔트리만 보존하고 새 프로세스의 엔트리만 추가한다.
d) TLB 크기를 동적으로 조절한다.

(참조: 9장 페이징 메모리 관리.pdf, p.34)

10.탐구 문제 (계산)

a) 프로세스 A는 26개의 페이지를 사용한다.
b) 물리 메모리의 프레임 개수는 500 000개이다.
c) 페이지 테이블 크기는 8 MB이다.
d) 내부 단편화 평균은 2 KB이다.

(참조: 9장 페이징 메모리 관리.pdf, p.15)
